## 2. Show HN
**Title:** `Show HN: CampaignOS – Self-hosted marketing automation powered by a local LLM`

I got tired of paying SaaS subscriptions to schedule posts and "AI-generate" copy that
just round-trips my data through someone's cloud. So I built CampaignOS: a self-hosted
marketing OS that does the whole loop — research a campaign, generate the content and
social copy with a *local* model via Ollama, schedule it, and broadcast to Telegram
channels — without any external API or per-token cost.

It's a single Python process: an aiogram Telegram bot + a FastAPI dashboard + an
in-process scheduler (one-off and recurring cron), persisting to SQLite. You point it at
your own Ollama instance (defaults to llama3.1:8b), add the bot to your channels, and go.
Nothing leaves your machine.

It's early but working — the end-to-end path (generate copy → connect channel → schedule
→ broadcast) is wired and tested. MIT licensed. I'd love feedback on the architecture and
on what would make you actually switch off a hosted tool.

## 4. r/selfhosted
**Title:** `CampaignOS: self-hosted marketing automation with a local LLM (Telegram + web dashboard, no SaaS, no cloud AI)`

Sharing a thing I've been building for the self-hosted crowd. CampaignOS runs your
marketing campaigns end-to-end on your own hardware: plan a campaign, generate content and
social copy with a **local** model (Ollama), schedule posts, and broadcast to Telegram
channels. No external API, no subscription, no data leaving the box.

- **One process:** aiogram Telegram bot + FastAPI web dashboard + in-process scheduler (one-off + recurring cron)
- **Local AI:** defaults to llama3.1:8b via Ollama — swap in any model you've pulled
- **State:** plain SQLite file
- **License:** MIT

`make setup && make model && python main.py` and you've got a dashboard on :8000. Add the
bot to a channel as admin to broadcast.
